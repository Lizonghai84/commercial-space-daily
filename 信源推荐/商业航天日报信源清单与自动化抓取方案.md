# 商业航天日报信源清单与自动化抓取方案

## TL;DR
- **核心结论**：建议以"RSS 聚合（SpaceNews / NASA / ESA / Spaceflight Now）+ The Space Devs Launch Library 2 API（发射）+ CelesTrak/Space-Track GP 数据（在轨）+ arXiv API（论文）+ 自建 RSSHub（中文社媒/公众号）+ 少量针对性爬虫（中文媒体与公司官网）"为骨架；中文侧最值得每日必抓的是国家航天局、卫星百科（灰机 wiki，开放标准 MediaWiki API）、SpaceLive/CHXLive 发射追踪、泰伯网与小火箭等。
- **用户候选源评估**：SpaceNews（强，有官方 RSS /feed）、Payload（强，但以邮件 newsletter 为主、无公开 RSS）、SatNews（中，页面有 RSS）、nextspaceflight（发射时刻表，无官方 API，建议改用其上游 Launch Library 2 API）、nasa.gov（强，多条官方 RSS）、satellitemap.space / n2yo / spacestatsonline（数据可视化型，作补充；n2yo 有官方 REST API）、arXiv（强，有官方 Atom API）。
- **落地建议**：4 人 Python 团队用"FreshRSS/自建 RSSHub 做聚合层 + 定时 Python 任务调 LL2 / arXiv / CelesTrak / 卫星百科 API + Playwright 针对 5–8 个中文站做定向爬虫"即可在 1–2 周内跑通 MVP；中文公众号/微博抓取是最大不稳定点，需做容错与多镜像。

## Key Findings
1. **英文权威媒体普遍提供 RSS**，自动化成本最低：SpaceNews（spacenews.com/feed）、NASA（多条官方 RSS）、ESA（官方 RSS）、Spaceflight Now（/feed）都开箱即用。SpaceNews 为行业权威，创刊号为 **1989 年 9 月 18 日**，总部弗吉尼亚州 Alexandria，纸刊每年出 12 期，网站受众逾 50 万航天从业者与爱好者。
2. **发射数据有一个"事实标准"免费 API**：The Space Devs 的 Launch Library 2（LL2）。据其官方 FAQ，"全部数据免费访问，**每 IP 每小时 15 次调用**"；开发可用 `lldev` 前缀（无限流但数据陈旧、仅供测试）。覆盖全球轨道发射含中国民商火箭；用户候选源 nextspaceflight 即由该团队运营，建议直接对接 API 而非爬网页。
3. **在轨/轨道数据的权威免费来源是 CelesTrak + Space-Track**（均源自美国太空军第 18 太空防御中队 GP/SGP4 数据）。**关键风险**：CelesTrak 官网置顶警告，5 位 NORAD 编号将在 **69999（非 99999）耗尽，预计约 2026-07-12**（撰稿时已到 69328）；届时新目标为 100000+ 编号，TLE 旧格式无法承载，须迁移到 2020 年 5 月起提供的 GP/OMM（XML/KVN/JSON/CSV）格式。
4. **中文侧的结构化"金矿"是卫星百科（灰机 wiki）**：2020 年成立、托管于灰机 wiki，首页显示现有 **19,011 个词条**；灰机平台基于 MediaWiki（1.38.4，2022-11-15 由 1.30 升级），官方声明"支持全部 MediaWiki API"。可直接经 `sat.huijiwiki.com/api.php` query 发射记录、卫星词条，是中文最易自动化的源。
5. **中文社媒/公众号必须靠自建 RSSHub**：知乎、微博、B站、微信公众号官方都不提供 RSS，RSSHub 可生成，但微博/知乎反爬严重、公众号尤其不稳定（第三方源"三天两头失效"），需自建实例 + 多镜像容错。
6. **部分中国商业公司官网可爬、部分只能靠国家媒体**：长光卫星（jl1.cn）、蓝箭航天（landspace.com）官网为静态 HTML、文章 ID 连续，最易爬；中国星网（GW/国网）、上海垣信（千帆）无可爬的一手新闻源，只能依赖新华社/央视与发射追踪站。

## Recommendations

**分层抓取方案（核心每日必抓 vs 补充）**

- **第一层｜核心每日必抓（自动化、稳定）**
  1. 英文新闻：SpaceNews RSS、Spaceflight Now RSS、SpaceDaily RSS、SNAPI 聚合 API（一次拿多家）、Ars Technica Space RSS。
  2. 发射时刻表：Launch Library 2 API（每日拉 upcoming + 过去 24h）。
  3. 中文官方：CNSA 要闻、CASC（RSS+爬）、中国载人航天网。
  4. 中文发射+资讯：SpaceLive（spacelive.cn）、卫星百科 MediaWiki API（拉最新发射记录）。
  5. 论文：arXiv API（astro-ph.IM / physics.space-ph / eess.SP 每日增量）。

- **第二层｜每日补充（半自动）**
  - 泰伯网、36氪航天标签、证券时报星探（中文产业/融资）。
  - CHXLive 发射日历（headless 渲染）。
  - 公司官网增量：长光卫星、蓝箭航天、星河动力、银河航天（ID 连续，轻量爬虫）。
  - KOL 精选：小火箭、太空精酿、Dongfang Hour、Eric Berger、Jonathan McDowell（RSSHub 订阅 + 人工筛）。

- **第三层｜数据型按需查询（不必每日全量）**
  - CelesTrak / Space-Track GP 数据（关注千帆、GW、星链入轨批次时查询）。
  - satellitemap.space、n2yo（可视化/过境，做专题时用）。
  - Jonathan's Space Report / GCAT、Gunter's Space Page（校核与背景资料）。

**轻量级自动化技术方案（4 人 Python 团队，1–2 周 MVP）**

1. **聚合层**：自建 **RSSHub**（Docker 部署）+ **FreshRSS**/Tiny Tiny RSS 做统一 RSS 收口。RSSHub 负责把知乎、微博、B站、微信公众号、X 转成 RSS。
2. **API 抓取层**：Python 定时任务（APScheduler/Celery + cron）调用：LL2（发射）、arXiv（论文）、CelesTrak gp.php（在轨）、n2yo（过境，按需）、卫星百科 api.php（中文发射记录）、SNAPI（英文新闻聚合）。统一写入 PostgreSQL。
3. **定向爬虫层**：用 **requests + parsel/BeautifulSoup** 爬静态站（CNSA、长光、蓝箭、SpaceLive、泰伯）；用 **Playwright** 处理 JS 渲染站（东方空间、CHXLive）。文章 ID 连续的站做增量枚举。
4. **去重与分板块**：用标题+URL 指纹去重；用关键词/规则（或小型 LLM）自动归类到"行业动态/最新技术/研究成果/最新发射"四板块。
5. **稳定性**：RSSHub 配多公共镜像 + 自建实例做 fallback；公众号/微博抓取设失败重试与告警；CelesTrak 提前切到 GP/OMM 新格式以应对约 2026-07 的 5 位编号耗尽。
6. **产出**：每日定时生成 Markdown/HTML 日报，人工 10 分钟终审 KOL 与深度稿。

**会改变建议的阈值/基准**
- 若团队无力维护 RSSHub 反爬（微博/公众号频繁失效），应降级为"只抓官方+英文 RSS+API"，公众号改人工。
- 若需要严肃在轨入轨核验，应升级 Space-Track 账号并以其为准（CelesTrak 为镜像便捷源）。
- 若发射时刻表调用量超 15 次/小时，应申请 LL2 API key 或自建缓存层。

## Caveats
- **中文社媒/公众号抓取本质不稳定**：微信公众号无开放 Web 接口且反爬严格，RSSHub 第三方公众号源"三天两头失效"；微博、知乎已加反爬限制，官方 RSSHub 源常断，需自建并接受一定失败率。
- **航天爱好者网（spaceflightfans.cn）状态降级，不宜作一手源**：2023 年 6 月该网微博公告"停止国内航天发射预报"，并删除中国发射预报与历史数据（同期国内多个平台亦转为只发国外内容）；2025 年底虽有"恢复访问"公告，但 2026 年实测主站返回 nginx 默认页（"Welcome to nginx!"）或陈旧内容；其"航天爱好者"公众号/微博仍活跃但以国外内容为主。
- **CelesTrak 编号过渡风险**：5 位 NORAD 编号预计 **2026-07-12 前后在 69999 耗尽**（非 99999），TLE 旧格式将无法承载新目标（100000+），须迁移到 GP/OMM（JSON/XML/CSV）。
- **中国星网（GW）与上海垣信（千帆）缺一手可爬源**：相关发射须依赖新华社/央视通稿与发射追踪站；spacesail.net 是否为垣信现行官网仍需人工核实。
- **数据准确性**：satellitemap.space、findstarlink 等为爱好者项目（非 SpaceX 官方），数据来自 Space-Track 每 12 小时刷新，存在延迟；统计口径以 Jonathan McDowell / Space-Track 为权威基准。
- **部分英文深度内容付费**：SpaceNews、Payload Pro、NASASpaceflight 论坛有付费墙，免费层足够日报选题但深稿需订阅。
- **时间敏感性**：本清单中的发射实例、在轨数量等为 2026 年 6 月初快照，会随时间变化；社媒账号名与公众号 id 亦可能变动，上线前建议逐一复核。
