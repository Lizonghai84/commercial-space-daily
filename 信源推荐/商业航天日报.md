日报的信源不要只按“媒体列表”做，最好按**信息链路**做：  

**官方/监管 → 发射与轨道数据库 → 行业媒体 → 公司官方 → KOL/视频 → 论文/研究**。这样可以交叉验证，避免只被新闻稿或二手媒体带偏。

下面是建议的第一版信源池。

## 一、最高优先级：结构化数据源，适合自动抓取

|信源|适合抓什么|抓取方式|可信度|建议用途|
|---|---|--:|--:|---|
|**The Space Devs / Launch Library 2**|全球发射计划、发射状态、运载火箭、任务、发射场、航天事件|免费 API|高|做“最新发射/即将发射”的主数据源|
|**Spaceflight News API / SNAPI**|航天新闻聚合，包括 SpaceNews、NASA、Ars 等来源|免费 API|中高|做新闻发现，不做最终事实源|
|**CelesTrak**|TLE/GP 轨道数据、SATCAT、活跃卫星、星座分类|静态文件/API式下载|很高|做“卫星入轨后验证”“星座数量变化”|
|**Space-Track.org**|美国官方空间目标目录、TLE、再入、空间态势数据|需注册 API|很高|做轨道验证、入轨确认、再入跟踪|
|**N2YO**|卫星位置、TLE、过境预测|免费但限额 API|中高|做前端可视化、单星跟踪|
|**Next Spaceflight**|发射日程、任务页、发射提醒|页面可抓；免费 API 目前偏有限/需联系|高|做发射预告补充|
|**RocketLaunch.Live**|全球发射日程、倒计时、发射状态|页面抓取|中高|发射预告交叉验证|

Launch Library 2 明确提供免费开放的发射与航天事件 API，适合作为日报的“发射日历底座”。([TheSpaceDevs](https://thespacedevs.com/llapi?utm_source=chatgpt.com "TheSpaceDevs - LL2 API")) Spaceflight News API 是 The Space Devs 提供的新闻 API，可作为新闻发现层。([航天新闻API](https://api.spaceflightnewsapi.net/v4/docs/?utm_source=chatgpt.com "Spaceflight News API 4.30.2 (v4) OAS 3.0")) CelesTrak 提供 GP/TLE、SATCAT 等轨道与卫星目录数据，而且其文档已经提醒开发者注意 5 位 NORAD 编号即将耗尽、推荐使用新格式，这一点对长期系统设计很重要。([CelesTrak](https://celestrak.org/NORAD/documentation/gp-data-formats.php?utm_source=chatgpt.com "GP data documentation")) Space-Track.org 提供 REST API，可用稳定 URL 和参数程序化获取与网页一致的数据。([Space-Track](https://www.space-track.org/documentation?utm_source=chatgpt.com "Help Documentation")) N2YO 提供 REST API，免费但按接口类型有调用限额。([N2YO](https://www.n2yo.com/api/?utm_source=chatgpt.com "N2YO.com API"))

**建议：**  
如果要做 MVP，发射数据先用 **Launch Library 2 + Next Spaceflight/RocketLaunch.Live 交叉验证**；入轨验证用 **CelesTrak + Space-Track**；卫星可视化再接 **N2YO 或 satellitemap.space**。

## 二、官方与监管信源：权威，但不一定及时

|信源|覆盖范围|抓取方式|价值|
|---|---|--:|---|
|**NASA.gov / NASA RSS**|NASA 商业载人、CLPS、ISS、Artemis、SpaceX 合作任务|RSS|官方确认源|
|**ESA RSS**|欧洲发射、Ariane/Vega、欧洲商业航天政策|RSS|欧洲官方源|
|**FAA Commercial Space Transportation**|美国商业发射许可、发射安全、事故调查|页面/公告抓取|SpaceX、Blue Origin、Rocket Lab 美国发射监管|
|**FCC Space Bureau**|星座频谱、卫星通信许可、Starlink/Kuiper/AST 等申请|页面/公开文件|卫星互联网政策与牌照|
|**CNSA / 国家航天局英文站、中文站**|中国官方航天新闻|页面抓取|中国官方确认源|
|**中国载人航天、CASC、CASIC、海南商业航天发射场**|中国发射任务、火箭、发射场动态|页面/公众号|中国发射一手源|
|**ITU / IFIC / 频轨申报信息**|星座频率、轨道资源、国际申报|文件/半结构化|判断星座规划真实性|

NASA 明确提供 RSS，可直接用于自动订阅。([NASA](https://www.nasa.gov/rss-feeds/?utm_source=chatgpt.com "NASA RSS Feeds")) ESA 也提供多个 RSS feed，覆盖新闻和项目更新。([欧洲航天局](https://www.esa.int/Services/RSS_Feeds?utm_source=chatgpt.com "ESA - RSS Feeds")) FAA 商业航天办公室负责美国商业发射与再入的安全、许可和监管。([联邦航空管理局](https://www.faa.gov/space?utm_source=chatgpt.com "Commercial Space Transportation")) FCC Space Bureau 负责卫星与天基通信相关政策和许可。([联邦通信委员会](https://www.fcc.gov/space?utm_source=chatgpt.com "Space | Federal Communications Commission")) CNSA 英文站会发布中国发射与任务新闻，适合作为中国官方确认源。([中国国家航天局](https://www.cnsa.gov.cn/english/?utm_source=chatgpt.com "CNSA"))

**注意：** 官方源通常“准但慢”，尤其是中国商业航天，很多信息先出现在新华社、央视、地方政府、公司公众号，再被英文媒体整理。所以官方源适合做**最终确认**，不一定适合做**第一时间发现**。

**中国信源抓取建议：**

2. 公众号文章通过第三方聚合/RSSHub/自建搜狗微信搜索抓取，但要注意稳定性和合规；
    
3. 新华社、央视、中国日报、澎湃、财联社、科创板日报做二级确认；
    
4. CelesTrak/Space-Track 做“是否真的入轨”的最终验证。

## 九、技术抓取建议

第一版不要一开始就做复杂爬虫，建议按这个顺序：

**第一阶段：RSS/API 优先**

- RSS：NASA、ESA、部分媒体、arXiv；
    
- API：Launch Library 2、SNAPI、CelesTrak、Space-Track、N2YO、Semantic Scholar、Crossref；
    
- 简单网页抓取：公司新闻页、Spaceflight Now、RocketLaunch.Live、CNSA。
    

**第二阶段：关键词过滤 + 实体识别**

建议维护一张实体表：

- 公司：SpaceX、Blue Origin、Rocket Lab、ULA、Firefly、Relativity、Stoke、Axiom、AST SpaceMobile、Planet、Maxar、Amazon Kuiper、Eutelsat OneWeb、星网、垣信、蓝箭、星河动力、天兵、中科宇航、东方空间；
    
- 星座：Starlink、Kuiper、OneWeb、Qianfan/千帆、Guowang/GW、IRIS²；
    
- 火箭：Falcon 9、Starship、New Glenn、Electron、Neutron、Vulcan、Ariane 6、Vega-C、长征八号、长征十二号、朱雀、谷神星、天龙、力箭、引力一号；
    
- 事件：launch、orbit insertion、deorbit、reentry、static fire、hot fire、landing test、license、contract、funding、spectrum、filing。
    

**第三阶段：事实校验**

每条日报建议至少标注：

- `source_type`: official / media / database / social / paper
    
- `confidence`: high / medium / low
    
- `verification`: single-source / cross-checked / orbit-confirmed
    
- `event_time`: UTC + 北京时间
    
- `companies`: 标准化实体
    
- `mission`: 标准化任务名
    
- `launch_vehicle`: 标准化火箭
    
- `payloads`: 卫星/载荷
    
- `orbit`: LEO/MEO/GEO/SSO/GTO 等
    
- `norad_id`: 如果已入轨并可查到
